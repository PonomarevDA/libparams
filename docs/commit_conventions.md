# Commit Conventions

Use short imperative commit messages: `type: summary`

Recommended types:

- feat: user-visible feature or new scenario
- fix: bug fix or behavior correction
- docs: documentation, notes, LICENSE, README
- test: tests, test data, test helpers
- refactor: code restructuring without intended behavior change
- ci: CI workflow, checks, or automation configuration
- chore: build scripts, tooling, dependencies, repo maintenance
- wip: temporary local checkpoint, should not normally reach shared history
